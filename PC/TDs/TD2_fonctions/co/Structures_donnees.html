def racines_trinome(a, b, c) :
    d = b ** 2 - 4 * a * c
    if d < 0 : return( complex(-b, - pow(-d, 0.5)) / (2 * a), \
		                   complex(-b, pow(-d, 0.5)) / (2 * a) )
    if d == 0 : return(-b / (2 * a))
    if d > 0 : return((-b - pow(d, 0.5)) / (2 * a), (-b + pow(d, 0.5)) / (2 * a))

Question

Cet algorithme n'est pas fonctionnel. En préciser la raison et proposer une correction.

Solution

Le problème est ici pour le test concernant le discriminant nul. Utiliser un test d'égalité n'a pas de sens concret entre deux flottants en raison des erreurs d'arrondis.

Il faut donc utiliser un test vérifiant si la différence entre les deux flottants est inférieure à une valeur fixée.

1
if abs(d) < eps : return(-b / (2 * a))
if abs(d) < eps : return(-b / (2 * a))

Ce test doit être le premier, avant les tests de signe du discriminant sinon, s'il est négatif et inférieur à , la fonction renverra les deux racines complexes.

Il faut aussi affecter la valeur de eps, selon l'utilisation de votre programme et de votre machine.

Par exemple :

1
eps = 1e-12
eps = 1e-12

    nombre = ""
    q, r = n // 2, n % 2
    while (q, r) != (0, 0) :
        nombre = str(r) + nombre
        q, r = q // 2, q % 2

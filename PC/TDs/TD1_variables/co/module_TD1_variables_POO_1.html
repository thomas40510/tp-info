Introduction

Bien programmer avec Python !

Avant tout voici quelques règles de bonne présentation d'un programme en Python afin de faciliter la lecture et la relecture :

  • Placer une espace après une virgule, un point-virgule, deux points.

  • Ne pas placer d'espace avant une virgule, un point-virgule, deux points.

  • Placer une espace de chaque côté d'un opérateur.

  • Ne pas placer d'espace entre le nom d'une fonction et sa liste d'arguments.

  • Indenter, conformément à la syntaxe de Python, les instructions successives. Pour cela, faites confiance à l'éditeur ; lorsque vous passez à la ligne, l'indentation doit être correcte, sinon, c'est que vous avez généralement omis de fermer une parenthèse ou fait une erreur de syntaxe sur les lignes précédentes.

Autre conseil important, l'informatique pour tous donne lieu à une épreuve écrite lors des concours, il faut donc s'habituer le plus tôt possible à écrire vos lignes de codes sur feuille.

Je vous conseille donc de toujours travailler avec une feuille, ou même un cahier sur lequel vous écrivez d'abord vos lignes de codes que, seulement après, vous testerez sur machine. L'ordinateur sert alors de correcteur, et vous pouvez noter sur votre feuille ou cahier les erreurs classiques que vous faites.

Votre progression sera alors fulgurante !

Démarrage de Python

Lancer l'environnement de travail Pyzo depuis le raccourci présent sur le bureau dans l'onglet Maths.

Il est à présent possible d'entrer une instruction dans la console qui est la fenêtre Shells, puis de l'exécuter en appuyant sur entrée.

Il est aussi possible de sauvegarder ses instructions dans un fichier dans la fenêtre voisine et d'exécuter votre script via le menu Run.

Pensez alors à sauvegarder régulièrement votre travail via le menu File/Save dans votre répertoire Mes documents. Vous pouvez y créer un sous-répertoire pour y sauvegarder le travail que vous allez faire durant les séances de TD d'informatique.

Si vous souhaitez conserver ce sujet de TD, il vous suffit de récupérer le fichier TD1.zip, et de l'extraire chez vous.

À la fin de la séance, après avoir quitté Pyzo, n'oubliez pas de fermer votre session.

Ce TD fait suite au « cours 1 : Rappels et Compléments sur les structures de données » et ses exemples d'application.

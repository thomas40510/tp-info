Gestion des variables dans la console

Voici quelques exemples à tester, d'abord en anticipant leurs résultats à l'écrit, puis en les testant avec Python. Commenter ces essais sur votre compte rendu.

Question
1
## Exemple 3
2
L_1 = [1, 2, 3, 4]; L_2 = L_1; L_2[0] = 'a'
3
print(L_1, L_2)
## Exemple 3
L_1 = [1, 2, 3, 4]; L_2 = L_1; L_2[0] = 'a'
print(L_1, L_2)
À l'issue de cet exemple, créer une variable contenant 'abc' et tester l'égalité de cette variable et de L_1[0] += 'bc', avec les tests == et is.
Solution
1
['a', 2, 3, 4] ['a', 2, 3, 4]
['a', 2, 3, 4] ['a', 2, 3, 4]
L2 est un alias de L1, ce nom de variable pointe donc vers la même liste que L1. Puis, l'un des termes de la liste mémorisée est modifiée, les deux variables qui sont restées des alias pointent alors vers cette liste modifiée.
Les exemples suivants cherchent à modifier une liste existante.

Question
1
## Exemple 6
2
L = [1, 2, 3, 4]; print(L, id(L))
3
L = [val + 1 for val in L] ; print(L, id(L))
## Exemple 6
L = [1, 2, 3, 4]; print(L, id(L))
L = [val + 1 for val in L] ; print(L, id(L))
Solution
1
[1, 2, 3, 4] 2021523329864
2
[2, 3, 4, 5] 2021523362312
[1, 2, 3, 4] 2021523329864
[2, 3, 4, 5] 2021523362312
Python a donc créé une nouvelle liste, c'est à dire que la liste précédente n'est plus pointée par la variable L. C'est le garbage collector qui libérera cette place mémoire.

Indice
On peut légitimement s'interroger sur la méthode la plus rapide entre ces deux derniers exemples. Il est possible de le faire en utilisant le module time.
Solution
1
[1, 2, 3, 4] 2021523332488
2
[2, 3, 4, 5] 2021523332488
[1, 2, 3, 4] 2021523332488
[2, 3, 4, 5] 2021523332488
La liste conserve son adresse, mais ses valeurs sont modifiées. Cette méthode est donc moins gourmande en espace mémoire et ne fait pas travailler le garbage collector.

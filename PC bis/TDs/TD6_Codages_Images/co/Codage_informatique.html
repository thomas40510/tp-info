Codage informatique

Le codage informatique des caractères est une convention qui permet, à travers un codage connu de tous, de transmettre de l'information textuelle.

Le codage ASCII

Le plus ancien code informatique est le code ASCII (American Standard Code for Information Interchange) qui définit un jeu de 128 caractères numérotés de 0 à 127. Ce code permet donc d'écrire de l'anglais ou toute autre langue utilisant l'alphabet latin sans les caractères accentués.

Le codage ISO 8859-n

Rapidement, le besoin de nouveaux caractères se fait sentir, et des extensions du code ASCII apparaissent, comme la famille des codes ISO 8859 au nombre de 16 et désignés par ISO 8859-n où n est un entier compris entre 1 et 16.

Par exemple, le code correspondant aux caractères nécessaires pour écrire le français est le ISO 8859-1 (parfois appelé Latin-1) et ISO 8859-2 permettra d'écrire le croate.

Tous ces codes définissent chacun un jeu de 224 caractères codés de 0 à 127 et de 160 à 255 (les codes dans l'intervalle ne sont pas utilisés). Un seul octet est donc suffisant pour les représenter.

Le standard Unicode

Le problème avec le codage ISO 8859-n est que deux codes différents ne seront pas compatibles car les caractères codés de 160 à 255 ne seront pas les mêmes. De plus, il n'est pas possible de transcrire toutes les écritures humaines avec ce code.

C'est pourquoi Unicode a vu le jour au début des années 1990 (première publication en octobre 1991) afin de standardiser les échanges de textes dans les différentes langues à un niveau mondial. La dernière version Unicode 11.0 a été publiée en juin 2018 et comporte caractères.

Ce standard attribue à chacun des caractères un nom et un numéro appelé point de code. Il est le plus souvent exprimé sous la forme U+XXXX (XXXX en base hexadécimale).

Exemple : le caractère 'A' a pour nom 'LATIN CAPITAL LETTER A' et pour code U+0041, son point de code est donc 65 en décimal.

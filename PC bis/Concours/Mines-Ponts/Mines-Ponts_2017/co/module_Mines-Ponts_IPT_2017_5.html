def versFile(n, taille):
    L = [bool(int(elt)) for elt in list(bin(n))[2:]]
    nL = len(L)
    if nL <= taille:
        return [False] * (taille - nL) + L

def init_files(n1 = 11, n2 = 11, L1 = '', L2 = '', p1 = 0.5, p2 = 0.3):
    '''Initialisation d'un croisement de deux files de longueurs impaires en leurs milieux. La file L1 est prioritaire sur la file L2.
    Les files sont décrites en argument par des nombres binaires.
    Par défaut, les longueurs des piles sont de 11 et elles sont dans une situation aléatoire. 
    L'apparition d'une voiture sur chaque file suit une loi de Bernoulli de paramètre p, par défaut de 0.5 sur la file 1, et de 0.3 sur la file 2.'''
    m1, m2 = n1 // 2, n2 // 2
    if L1 == '':
        L1 = [bool(elt) for elt in rd.binomial(1, p1, n1)]
    elif L1 != '':
        L1 = versFile(int(L1, 2), n1)
    if L2 == '':
        L2 = [bool(elt) for elt in rd.binomial(1, p2, n2)]
        if L1[m1]:
            L2[m2] = False
    elif L2 != '':
        L2 = versFile(int(L2, 2), n2)
    return([L1, L2])

Enregistrer / Créer une animation au format .gif

Pour celles et ceux que cela amuse bien sûr, c'est une activité totalement hors de notre programme.

Il est possible de créer une animation avec un logiciel libre : ImageMagick.

Pour cela, il faut demander à Python de lancer une commande exécutable.

1
magick convert -delay 20 -loop 0 croisement_*.png  Anim_croisement.gif
magick convert -delay 20 -loop 0 croisement_*.png  Anim_croisement.gif

-loop 0 veut dire que l'on demande une boucle infinie. Un autre chiffre commande une boucle finie d'autant de cycles.

-delay 20 donne le délai entre deux images en centième de secondes.

Si vous souhaitez ajuster ce délai à celui de l'animation pixel, il faut penser l'adapter.

Tout cela se fait assez facilement avec le module os.

    FF = simul_files(N, F0, p1, p2)
    # FF contient la la liste de toutes les étapes de l'évolution du croisement
    
    m1, m2 = n1 // 2, n2 // 2
